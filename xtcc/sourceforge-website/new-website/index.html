QScript Overview

QScript is a language to help script questionnaires for market research for data collection. The language supports routing of questions using conditional logic, simple looping, masking of codes based on answers to previous questions and mutually exclusive codes.

Workflow Overview

  • A Data Processing executive prepares a qscript file according to the questionnaire logic.
  • Once you compile a script, you get an exe, which allows you to enter the questionnaire data.
  • She will then test it to ensure that it behaves correctly for various inputs and logic.
  • The data entry script can then be sent to locations where fieldwork is conducted, or a centralized Data Entry office where the data can be punched.
  • The exe saves data in simple .dat files, which the Data Entry teams can email back to the Data Processing team.
  • The data processing executive will use same exe program to write out the data as a flat ascii file, quantum data file, and xtcc data files according to the needs of the Data Processing software used
  • While writing out the flat data file, the exe will also generate frequency counts for each question. This can used in checking the delivered data/tables for correctness

Data Formats

The compiled exe, generated from the script will also generate the appropriate tabulation program/scripts.

  • While generating a quantum data file, it generates specs for the Quantum Data Processing language,
  • For the ascii data file, it generates an SPSS script
  • and for the xtcc data file it generates the programs for the xtcc Data Processing Language.

Current Use Case Scenario

Currently, qscript is good for scripting Pen and Paper Interview (PAPI) questionnaires. This would be applicable to countries where interviews are conducted in such a format, typically India and the Middle East.

An Example script

Here is a small script to give you a feel for the language

Example Script
inp_simple_example {

q1 "Q1. This is question 1" sp int32_t (5-8, 1-2, 9-10, 11, 16) ;


q2 "Q2. This is question 2" mp(5) int32_t (5-8, 1-2, 9-10, 11, 16);

// define a stubs variable which will be used throughtout a survey
stubs stuff =
"USB pen drive" 1
"Keys" 2
"Wallet" 3
"Mobile Phone" 4
"Pen" 5 ;

q3 "Q3. Which of these things are usually in your pocket, when on the move?"
        mp(3) int32_t stuff ;
unset (stuff);
setadd (stuff, q3);
q4 "Q4. And which of these irritate you the most when in your pocket?"
        mp(3) int32_t stuff ;


stubs agree5 =
" strongly disagree" 1
" disagree " 2
" neutral " 3
" agree " 4
"strongly agree" 5;

if (q1 in (1) ) {
        q2_1 "Q2.1 This is question 2_1" sp int32_t agree5;
} else {
        q2_2 "Q2.2 This is question 2_2" sp int32_t agree5;
}
}
Compiling a script

Compiling the above script. Suppose the file is named inp_simple_example. It can be compiled to an exe with the following commands

Compilation on GNU/Linux

				$ qscript -n  -m -o -f inp_simple_example

Compilation on Windows - you must pass the -s option to generate a static binary

				$ qscript -s -n  -m -o -f inp_simple_example

The -n flag is for a curses binary, the -m option creates a nice mapfile layout, the -o option is a optimization for save/restore jumps.

You can run the generated executable like this.

				$ .\inp_simple_example.exe

System Requirements

  • Runs on Windows/GNU Linux and should probably work on any UNIX like OS
  • Needs GNU g++ compiler on linux or MinGW on Windows
  • Needs ncurses library on linux or pdcurses library on Windows
