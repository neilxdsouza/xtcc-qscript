QScript could be useful in the following scenarios

  • You do pen and paper studies and are looking for a solution to program the data entry script according to questionnaire logic and looking for clean data.
  • You are doing a multi-country study and you are doing field work in another country. You want to match the map and layout with the one you already have and avoid having to recode the data. In this case you script the questionnaire and send the exe to the client for data entry.
  • You want to save data processing time and use the qax file produced by qscript for your DP setup.
  • You want to automatically create a nice layout for the mapfile - qscript automatically reserves blank spaces between questions for future extensions of code-lists etc.
  • You want to save costs - either yours or if you are doing field work for a client - the client costs.
